# 项目规则
原地开发： 禁止擅自增删改非必要现有结构、CSS 类名或文件。除明确要求外，不准添加渐变色。
运行环境： 若服务器已启动，严禁重复执行启动命令。
明确环境： 必须要明确本项目的环境依赖和版本，禁止采用废弃的技术栈。
代码安全： 强制执行 JSX 闭合检查；三元运算符必须完整；列表必须包含 key；注释仅允许使用 {/* */}
核心MVP：  禁止简单功能过度设计和复杂化，除非用户明确需求。
# 必须遵守
1. **不假设原则**：不假设任何自定义组件或文件存在
2. **询问优先**：对不确定的引用先询问或提供选项
3. **方案提供**：为可能缺失的文件提供创建代码
4. **路径安全**：优先使用相对路径，其次考虑别名
# 【文件组织预防措施】
1. 创建文件前先检查现有目录，禁止重复创建功能相同的文件
2. 组件文件必须放入/components目录，禁止散落在pages或其他目录
3. 页面文件必须放入/pages目录，禁止在app目录下直接创建页面
4. 工具函数必须放入/lib或/utils目录，禁止在组件文件内定义复杂工具函数
5. 类型定义必须放入/types目录或集中在types.ts，禁止分散在各文件中
6. 样式文件必须放入/styles目录，禁止在组件文件外零散创建.css文件
7. 每个目录下文件超过10个时必须创建子目录分类，禁止平铺过多文件
8. 文件命名必须体现所属模块，如auth/LoginBtn.tsx而非Btn.tsx
9. 创建新文件前必须确认同功能文件不存在，避免命名变体如utils.ts和helpers.ts共存
10. 私有文件使用下划线前缀（如_core、_utils），临时/备份文件使用_temp或_old后缀，禁止无标记的混杂文件混入正式文件
# 【命名简洁规则】
1. 组件文件名不超过15个字符（不含.tsx）
2. 使用标准缩写：Button→Btn、Input→Inp、Password→Pwd、Section→Sec
3. 避免重复描述：PasswordStrengthIndicator→PwdStrength
4. 通用组件保留全称：FormCard、NavBar（已简洁）
5. 索引文件统一用index.tsx，不添加额外描述
