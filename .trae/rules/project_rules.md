# 项目规则
原地开发： 禁止擅自增删改非必要现有结构、CSS 类名或文件。除明确要求外，不准添加渐变色。
运行环境： 若服务器已启动，严禁重复执行启动命令。
明确环境： 必须要明确本项目的环境依赖和版本，禁止采用废弃的技术栈。
代码安全： 强制执行 JSX 闭合检查；三元运算符必须完整；列表必须包含 key；注释仅允许使用 {/* */}
核心MVP： 用最简单可靠的方式解决问题，同时为未来留下可扩展的余地，但绝不提前复杂化
# 必须遵守
1. **不假设原则**：不假设任何自定义组件或文件存在
2. **询问优先**：对不确定的引用先询问或提供选项
3. **方案提供**：为可能缺失的文件提供创建代码
4. **路径安全**：优先使用相对路径，其次考虑别名
# 【文件组织预防措施】
1. 创建文件前先检查现有目录，禁止重复创建功能相同的文件
2. 组件文件必须放入/components目录，禁止散落在pages或其他目录
3. 工具函数必须放入/lib或/utils目录，禁止在组件文件内定义复杂工具函数
4. 类型定义必须放入/types目录或集中在types.ts，禁止分散在各文件中
5. 每个目录下文件超过10个时必须创建子目录分类，禁止平铺过多文件
6. 文件命名必须体现所属模块，如auth/LoginBtn.tsx而非Btn.tsx
7. 创建新文件前必须确认同功能文件不存在，避免命名变体如utils.ts和helpers.ts共存
8. 私有文件使用下划线前缀（如_core、_utils），临时/备份文件使用_temp或_old后缀，禁止无标记的混杂文件混入正式文件
# 【命名简洁规则】
1. 组件文件名不超过15个字符（不含.tsx）
2. 使用标准缩写：Button→Btn、Input→Inp、Password→Pwd、Section→Sec
