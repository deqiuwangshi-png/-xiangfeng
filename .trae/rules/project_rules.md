# 项目规则
不允许乱改我写的代码
禁止私自添加、删除、修改项目中的文件，除非用户特别明确开发范围
禁止删除项目中的文件，除非用户特别明确删除范围
禁止修改我设计好的HTML文件
不需要运行Next服务器，除非这个服务器已经运行了不需要重复启动
我采用的是Tailwind CSS 4.1.18最新版本、Next.js 16.1.1、React 19.2.2版本
- 类型系统 : TypeScript 5
- 代码检查 : ESLint 9
先分析项目架构文件，确定项目的目录结构和文件组织方式，避免重复性创建开发文件
利用现有的架构合理组织项目文件，避免文件冗余和冲突。
- 组件目录 : 所有React组件都应放在`/components`目录下，每个组件有一个单独的文件。
- 页面目录 : 所有Next.js页面都应放在`/pages`目录下，每个页面有一个单独的文件。
- 样式目录 : 所有Tailwind CSS样式都应放在`/styles`目录下，每个样式文件有一个单独的文件。
- 公共资源目录 : 所有公共资源（如图片、字体等）都应放在`/public`目录下。
- 其他目录 : 其他项目相关目录（如`/src`、`/app`等）根据需要组织。
在 React 中，HTML 属性需要遵循 camelCase 命名规范，而不是使用短横线命名法。例如，`class` 应写为 `className`，`for` 应写为 `htmlFor`。
