## 修复计划

### 1. 问题分析
- **错误信息**：`Unexpected token '<', "<!DOCTYPE "... is not valid JSON`
- **根本原因**：缺少处理单个文章请求的API路由 `/app/api/articles/[id]/route.ts`
- **具体问题**：
  - 当请求 `/api/articles/[id]` 时，由于路由不存在，Next.js 返回 HTML 404 页面
  - 前端代码尝试将 HTML 响应解析为 JSON，导致解析失败
  - API 工具函数缺乏健壮的错误处理，无法处理非 JSON 响应

### 2. 修复方案

#### 2.1 创建单个文章API路由
- **文件路径**：`/app/api/articles/[id]/route.ts`
- **功能**：处理单个文章的GET请求，根据ID返回文章详情
- **实现逻辑**：
  - 从请求URL中提取文章ID
  - 根据ID查找模拟文章数据
  - 若找到则返回文章详情（JSON格式）
  - 若未找到则返回404错误（JSON格式）

#### 2.2 优化API工具函数的错误处理
- **文件路径**：`/src/lib/api.ts`
- **功能**：增强`apiRequest`函数的错误处理能力
- **实现逻辑**：
  - 在调用`response.json()`前检查响应是否为JSON格式
  - 若响应不是JSON格式，返回更具体的错误信息
  - 优化错误处理，确保函数始终返回预期的`ApiResponse`格式

### 3. 修复步骤

1. **创建单个文章API路由**：
   - 创建`/app/api/articles/[id]/route.ts`文件
   - 实现GET方法，根据ID返回文章详情
   - 确保返回正确的JSON格式响应

2. **优化API工具函数**：
   - 更新`apiRequest`函数，添加响应类型检查
   - 确保只在响应是JSON时才尝试解析
   - 返回更具体的错误信息

3. **测试修复效果**：
   - 访问文章详情页，验证API请求成功
   - 检查控制台，确认不再出现JSON解析错误
   - 验证文章内容正确显示

### 4. 预期结果
- 不再出现`Unexpected token '<'`错误
- 文章详情页能正常加载和显示
- API请求失败时显示友好的错误信息
- 代码具有更强的健壮性，能处理各种异常情况
