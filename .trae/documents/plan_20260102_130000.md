## 修复导航逻辑bug的详细计划

### 问题分析

1. **路由结构**：
   - 着陆页（官网）：`/(landing)/page.tsx` - 对应默认路径 `/`
   - 应用首页：`/home/page.tsx` - 对应路径 `/home`
   - 探索页面：`/explore/page.tsx` - 对应路径 `/explore`
   - 发布页面：`/publish/page.tsx` - 对应路径 `/publish`

2. **bug原因**：
   - Sidebar组件中配置的首页导航链接是 `/`，而不是 `/home`
   - 当用户从发布页面返回并点击"首页"时，会跳转到着陆页（`/`），而不是应用首页（`/home`）
   - 这导致了用户体验的破坏，用户期望返回应用首页，却跳转到了官网着陆页

### 修复方案

**方案1（推荐）**：修改Sidebar组件中的导航链接配置，将首页的href从 `/` 改为 `/home`

**方案2**：调整路由结构，将应用首页移动到 `/` 路径，将着陆页移动到 `/landing` 路径

### 修复步骤

1. **修改Sidebar组件中的导航配置**：
   ```javascript
   // 当前配置（有问题）
   const navItems = [
     { id: 'home', label: '首页', icon: <Home ... />, href: '/', tab: 'home' },
     ...
   ];

   // 修复后的配置
   const navItems = [
     { id: 'home', label: '首页', icon: <Home ... />, href: '/home', tab: 'home' },
     ...
   ];
   ```

2. **验证修复效果**：
   - 确保从任意页面点击"首页"导航项都能正确跳转到 `/home`
   - 确保导航逻辑在所有页面间切换时都正常工作
   - 确保activeTab状态管理正确

### 修复后的预期效果

1. **从首页切换到探索页面**：正常导航到 `/explore`
2. **从探索页面切换到发布页面**：正常导航到 `/publish`
3. **从发布页面返回首页**：正常导航到 `/home`（应用首页），而不是着陆页（`/`）
4. **所有页面间的导航**：都能保持正确的应用内导航，避免异常跳转到官网着陆页

### 技术要点

- **用户体验设计**：确保导航行为符合用户预期
- **组件间通信**：确保activeTab状态在页面间正确传递和更新
