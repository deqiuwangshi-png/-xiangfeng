# 服务条款页面重构计划

## 1. 设计失误分析

### 1.1 导航系统
- **当前问题**：导航栏设计与设计稿不符，缺少返回按钮和移动端菜单
- **设计要求**：实现带有 Logo、导航链接、返回按钮和移动端菜单的完整导航栏

### 1.2 侧边导航
- **当前问题**：侧边导航过于简单，缺少激活状态、时间信息和相关文档
- **设计要求**：实现带有激活状态、最后更新时间、阅读时间和相关文档链接的侧边导航

### 1.3 内容布局
- **当前问题**：内容布局简单，缺少条款类型标签、丰富的标题样式和特殊区块
- **设计要求**：实现带有条款类型标签、多级标题样式、警告提示和重要条款卡片的复杂布局

### 1.4 组件化程度
- **当前问题**：几乎没有组件化，所有内容都在一个文件中
- **设计要求**：提取可复用组件，整合到组件库中

### 1.5 样式实现
- **当前问题**：样式简单，缺少自定义字体、阴影效果和动画
- **设计要求**：实现设计稿中的自定义样式、动画和阴影效果

## 2. 重构方案

### 2.1 提取共享组件

| 组件名称 | 描述 | 存放位置 |
|---------|------|----------|
| `NavBar` | 导航栏组件 | `src/components/layout/NavBar.tsx` |
| `SideNav` | 侧边导航组件 | `src/components/layout/SideNav.tsx` |
| `TermsSection` | 条款区块组件 | `src/components/ui/TermsSection.tsx` |
| `TermsWarning` | 警告提示组件 | `src/components/ui/TermsWarning.tsx` |
| `ImportantClause` | 重要条款卡片组件 | `src/components/ui/ImportantClause.tsx` |

| `ActionCard` | 行动卡片组件 | `src/components/ui/ActionCard.tsx` |
| `TermsTable` | 条款表格组件 | `src/components/ui/TermsTable.tsx` |

### 2.2 页面重构

1. **更新页面结构**：按照设计稿重构页面布局
2. **整合共享组件**：使用提取的组件重构页面内容
3. **实现响应式设计**：确保在不同设备上都能正常显示
4. **优化移动端体验**：实现移动端菜单和响应式布局

### 2.3 样式优化

1. **自定义字体**：实现设计稿中的字体样式
2. **自定义阴影**：实现设计稿中的阴影效果
3. **动画效果**：实现设计稿中的动画效果
4. **激活状态**：实现侧边导航的激活状态样式
5. **列表样式**：实现设计稿中的列表项样式

### 2.4 组件库整合

1. **遵循现有组件库设计模式**：按照现有组件库的设计模式实现新组件
2. **添加到组件库索引**：将新组件添加到组件库索引中
3. **确保可复用性**：设计通用的组件接口，确保组件的可复用性

## 3. 实现步骤

1. **分析现有组件库**：查看现有组件库的设计模式和实现方式
2. **提取可复用组件**：从设计页面中提取可复用的组件
3. **实现组件**：按照现有组件库的设计模式实现提取的组件
4. **重构页面**：使用实现的组件重构 terms 页面
5. **优化样式**：实现设计页面的样式效果
6. **测试响应式**：确保页面在不同设备上都能正常显示

## 4. 预期效果

1. **设计还原度**：95% 以上还原设计稿
2. **组件化程度**：实现高度组件化，提高代码复用性
3. **响应式设计**：适配各种设备尺寸
4. **性能优化**：优化组件渲染性能
5. **代码可维护性**：提高代码的可维护性和扩展性

## 5. 风险评估

1. **组件库整合风险**：确保新组件与现有组件库的设计模式一致
2. **样式实现风险**：确保自定义样式与现有样式系统兼容
3. **响应式设计风险**：确保在各种设备上都能正常显示
4. **性能风险**：避免组件过度渲染，确保性能优化
