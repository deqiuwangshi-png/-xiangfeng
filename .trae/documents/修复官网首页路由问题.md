## 问题分析

根据检查，官网首页未能正常显示的原因是路由冲突：

1. **路由规则冲突**：Next.js App Router 中，根路径 `/` 优先使用 `app/page.tsx`，而我们的官网首页在 `app/(landing)/page.tsx`
2. **路由组优先级**：路由组 `(landing)` 不会影响 URL 路径，但当根目录存在 `page.tsx` 时，会覆盖路由组内的 `page.tsx`
3. **默认页面冲突**：根目录下的 `app/page.tsx` 是 Next.js 默认生成的欢迎页面，导致官网首页被覆盖

## 解决方案

### 方案 1：删除冲突的默认页面（推荐）
删除根目录下的 `app/page.tsx` 文件，让 `app/(landing)/page.tsx` 成为根路径 `/` 的默认页面

### 方案 2：复制内容到根页面
将 `app/(landing)/page.tsx` 的内容复制到根目录下的 `app/page.tsx` 文件中

## 实现步骤

### 方案 1 实现步骤
1. 删除 `app/page.tsx` 文件
2. 保留 `app/(landing)/page.tsx` 作为官网首页
3. 更新 `app/layout.tsx` 中的元数据，使用正确的网站标题和描述

### 方案 2 实现步骤
1. 复制 `app/(landing)/page.tsx` 的全部内容
2. 替换 `app/page.tsx` 中的默认内容
3. 更新 `app/layout.tsx` 中的元数据

## 代码修改建议

### 更新 app/layout.tsx（两种方案都需要）
```tsx
// app/layout.tsx
import type { Metadata } from "next";
import { Noto_Sans_SC, Noto_Serif_SC } from "next/font/google";
import "./globals.css";

const notoSansSC = Noto_Sans_SC({
  variable: "--font-noto-sans-sc",
  subsets: ["latin"],
});

const notoSerifSC = Noto_Serif_SC({
  variable: "--font-noto-serif-sc",
  subsets: ["latin"],
});

export const metadata: Metadata = {
  title: "相逢 - 深度思考者的精神家园",
  description: "连接深度思考者，打破认知边界，构建属于你的思维网络",
};

export default function RootLayout({
  children,
}: Readonly<{
  children: React.ReactNode;
}>) {
  return (
    <html lang="zh-CN">
      <body
        className={`${notoSansSC.variable} ${notoSerifSC.variable} antialiased`}
      >
        {children}
      </body>
    </html>
  );
}
```

## 预期效果

修复后，访问根路径 `/` 时，会显示我们实现的官网首页，而不是 Next.js 默认的欢迎页面。路由组 `(landing)` 内的其他页面（如 `/about`、`/services` 等）也能正常访问。
