# Next.js 项目文件架构分析

## 项目概述
这是一个基于 Next.js 16.1.1 的 Web 项目，使用 TypeScript 和 Tailwind CSS 进行开发。项目名称为 "xiangfeng_web"，采用现代前端技术栈，是一个综合性的内容创作与社交平台。

## 技术栈说明

### 核心依赖（基于package.json确认）
- **Next.js 16.1.1**: React 全栈框架
- **React 19.2.3**: 前端 UI 库
- **React DOM 19.2.3**: React DOM 渲染器
- **TypeScript**: 类型安全的 JavaScript
- **Tailwind CSS 4**: 实用优先的 CSS 框架

### 开发依赖
- **ESLint 9**: 代码质量检查
- **PostCSS**: CSS 后处理器
- **@types/node**: Node.js 类型定义
- **@types/react**: React 类型定义
- **@types/react-dom**: React DOM 类型定义

### 开发工具
- **ESLint**: 代码质量检查
- **PostCSS**: CSS 后处理器

## 目录结构说明

### app 目录（App Router）
Next.js 13+ 引入的新路由系统，采用基于文件系统的路由：
- `layout.tsx`: 根布局组件，定义全局字体和元数据
- `page.tsx`: 首页组件，包含基本的欢迎界面
- `globals.css`: 全局样式，包含 Tailwind CSS 导入

### public 目录
存放静态资源文件，所有文件可通过 `/` 路径直接访问。


## 项目配置

### package.json 关键配置
- 开发脚本：`dev`（开发模式）、`build`（构建）、`start`（生产模式）、`lint`（代码检查）
- 依赖管理：包含 Next.js、React、TypeScript 等核心依赖

### 配置文件
- `next.config.ts`: Next.js 项目配置
- `tsconfig.json`: TypeScript 编译选项
- `eslint.config.mjs`: 代码规范检查配置
- `postcss.config.mjs`: CSS 处理配置

## 数据库架构

### 数据库技术栈
- **主数据库**: PostgreSQL 15+
- **ORM框架**: Prisma 5.x
- **缓存层**: Redis 7+
- **迁移工具**: Prisma Migrate
- **连接池**: Prisma Data Proxy

### 核心数据模型

#### 用户系统
- **User**: 用户基础信息表
- **UserSettings**: 用户设置表
- **UserProfile**: 用户扩展资料表
- **Follow**: 用户关注关系表

#### 内容系统
- **Article**: 文章主表
- **ArticleContent**: 文章内容表
- **Comment**: 评论表
- **Like**: 点赞记录表
- **Bookmark**: 收藏记录表
- **View**: 浏览记录表

#### 社区系统
- **Discussion**: 讨论主题表
- **DiscussionReply**: 讨论回复表
- **DiscussionLike**: 讨论点赞表
- **DiscussionView**: 讨论浏览表

#### 系统管理
- **SystemConfig**: 系统配置表
- **FileUpload**: 文件上传记录表
- **Notification**: 通知消息表
- **SystemLog**: 系统日志表

### 数据库配置
- **连接配置**: `DATABASE_URL` 环境变量
- **连接池**: 最大连接数 20，超时时间 30s
- **字符集**: UTF-8
- **时区**: UTC
- **备份策略**: 每日增量备份，每周完整备份
