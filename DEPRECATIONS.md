# Deprecated Features And Tables

本文档用于记录**已下线功能与对应数据库表**，作为团队协作的硬约束，避免后续误恢复已废弃能力。

## 1. 适用范围

1. 适用于本仓库所有前后端代码、数据库迁移、文档与设计评审。  
2. 任何涉及下列已下线模块的新增需求，必须先经过产品与技术评审。  
3. 未经评审，不得重新引入已下线表或同名能力。

## 2. 已下线模块清单（当前生效）

### 2.1 积分系统（已下线）

1. 功能范围：积分余额、积分流水、积分过期。  
2. 已删除表：

- `public.user_points`
- `public.point_transactions`
- `public.point_expiration`

### 2.2 任务系统（已下线）

1. 功能范围：任务配置、任务进度、任务领奖。  
2. 已删除/停用表：

- `public.tasks`
- `public.user_task_records`

### 2.3 商品/商城系统（已下线）

1. 功能范围：商品定义、库存、商城兑换。  
2. 已删除/停用表：

- `public.user_inventory`

### 2.4 历史兑换系统（已下线）

1. 功能范围：兑换记录、兑换发放状态追踪。  
2. 已删除/停用表：

- `public.exchange_records`

### 2.5 今日签到系统（已下线）

1. 功能范围：签到打卡、连续签到奖励。  
2. 已删除/停用表：

- `public.sign_in_records`

## 3. 代码约束（必须遵守）

1. 禁止在运行时代码中新增对上述表的 `.from('<table>')` 查询。  
2. 禁止新增 `types/rewards`、`icons/rewards` 相关模块与导出。  
3. 禁止新增“积分/任务/签到/商城/兑换”入口页面与路由。  
4. 文档中若提到上述能力，必须明确标注“已下线”，不得写成可用能力。

## 4. 数据库约束（必须遵守）

1. 不得新建与上述同名表（除非完成正式评审并附迁移方案）。  
2. 不得恢复这些表的 RLS 策略与触发器。  
3. 若历史 SQL 文件仍保留相关语句，仅作归档参考，不代表当前生效。

## 5. 变更流程（要恢复下线能力时）

1. 提交 RFC：说明业务必要性、成本、风险。  
2. 明确替代方案是否失败，以及为何不能复用现有能力。  
3. 提供完整迁移方案：

- 数据模型
- RLS 策略
- 回滚方案
- 前后端影响清单

1. 评审通过后再允许实施，不得“先做后补文档”。

## 6. 大白话说明（给新同学）

1. 这些功能已经关停了，不是“暂时不用”。  
2. 你看到旧文档或旧命名，不代表还能继续开发。  
3. 你要是直接把积分/任务代码接回来，线上很可能报表不存在或逻辑不一致。  
4. 拿不准就先问，不要猜。

## 7. 快速自检命令（提交前建议跑）

```bash
rg "user_points|point_transactions|point_expiration|tasks|user_task_records|exchange_records|user_inventory|sign_in_records" .
```

若出现运行时代码命中（非历史文档），请先清理再提交。
