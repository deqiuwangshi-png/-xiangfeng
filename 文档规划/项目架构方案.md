# 相逢 (Xiangfeng) 项目架构方案

## 项目概述

相逢是一个综合性的内容创作与社交平台，结合文章发布、用户互动、社区讨论等功能，旨在为用户提供高质量的内容体验和社交互动环境。

## 一、业务架构分析

### 1.1 核心业务模块

#### 内容管理模块
- **文章系统**：文章创作、编辑、发布、分类管理
- **内容展示**：文章列表、详情页、相关推荐
- **内容搜索**：全文搜索、标签搜索、分类筛选

#### 用户系统模块
- **用户认证**：注册、登录、第三方登录
- **个人中心**：个人信息管理、文章管理、收藏管理
- **用户互动**：关注、点赞、评论、分享

#### 社区互动模块
- **讨论区**：话题讨论、回复互动
- **探索功能**：内容发现、热门推荐
- **社交功能**：用户关系、消息通知

#### 系统管理模块
- **设置管理**：用户偏好、系统配置
- **内容审核**：文章审核、评论管理
- **数据分析**：用户行为分析、内容统计

### 1.2 用户角色与权限

#### 访客用户
- 浏览公开内容
- 搜索文章
- 查看用户资料

#### 注册用户
- 发布文章
- 参与讨论
- 关注用户
- 个人资料管理

#### 管理员
- 内容审核
- 用户管理
- 系统配置

## 二、技术架构设计

### 2.1 前端架构

#### 技术栈选择
```
框架：Next.js 16.1.1 (App Router)
语言：TypeScript 5
样式：Tailwind CSS 4
图标：Lucide Icons
字体：Noto Sans SC / Noto Serif SC
状态管理：React Context + Zustand
HTTP客户端：Axios
富文本：TipTap / Slate.js
```

### 2.2 组件架构

基于官网首页.html的组件分析，以下是详细的组件架构：

#### 基础组件层（UI Components）
**核心UI组件**（基于官网首页.html样式系统）：
- **Button**：支持渐变背景、悬停效果、按压动画
- **Input**：表单输入框，支持验证状态
- **Card**：卡片容器，包含悬停抬起效果和阴影扩散
- **Modal**：模态框，支持背景模糊和动画过渡
- **Toast**：消息提示，支持成功/错误/警告状态

**布局组件**（Layout Components）：
基于首页.html、探索.html、发布.html的共享布局分析，以下是核心布局组件：

**主应用布局组件**（Main Application Layout）：
- **MainLayout**：主应用布局容器，包含左侧边栏、顶部导航、主内容区域的三栏布局
  - 支持响应式设计：桌面端显示完整三栏，移动端可折叠侧边栏
  - 提供统一的间距系统和背景样式
  - 管理整体页面滚动和状态

- **LeftSidebar**：左侧边栏组件（基于首页.html、探索.html、发布.html共享侧边栏）
  - **用户头像区域**：显示用户头像、在线状态、用户名（仅桌面端显示）
  - **主导航菜单**：首页、探索、发布三个核心导航项
  - **品牌信息**：底部显示品牌标识和版本信息
  - **响应式适配**：移动端显示为图标模式，桌面端显示完整文字
  - **交互功能**：导航高亮、悬停效果、点击反馈

- **TopNavigation**：顶部导航栏组件
  - **页面标题**：显示当前页面名称
  - **操作按钮**：根据页面上下文显示相应操作（如发布页面的保存、预览按钮）
  - **面包屑导航**：支持多层级的页面导航
  - **响应式适配**：在移动端优化显示

- **RightSidebar**：右侧边栏组件（首页.html和探索.html共用）
  - **推荐内容区域**：显示热门文章、推荐用户、热门话题
  - **统计信息**：显示社区活跃度、用户参与度等数据
  - **快捷操作**：提供快速发布、快速搜索等入口
  - **响应式行为**：在较窄屏幕下自动隐藏或转换为底部区域

**官网着陆页布局组件**（Landing Page Layout）：
- **LandingLayout**：官网着陆页专用布局
  - **固定导航栏**：在页面滚动时保持顶部导航可见
  - **全屏区块设计**：支持hero区域的全屏展示
  - **锚点导航**：支持页面内区块间的平滑滚动跳转
  - **SEO优化**：提供完整的meta标签和结构化数据

**通用布局组件**（Common Layout Components）：
- **Container**：内容容器，提供一致的间距和最大宽度控制
  - **尺寸变体**：支持sm、md、lg、xl、full等多种宽度
  - **间距系统**：基于4px网格系统的内边距控制
  - **响应式断点**：适配不同屏幕尺寸的容器宽度

- **Section**：页面区块容器，支持背景渐变和装饰元素
  - **背景样式**：支持纯色、渐变、图案等多种背景
  - **装饰元素**：支持边角装饰、分割线、图标等视觉元素
  - **间距控制**：提供统一的上、下、内边距管理

- **Grid**：网格布局容器，用于卡片式内容展示
  - **响应式列数**：根据屏幕尺寸自动调整列数
  - **间距控制**：统一的网格项间距
  - **对齐方式**：支持start、center、end、stretch等对齐选项

**数据展示组件**（Data Display）：
- **ArticleCard**：文章卡片，支持封面图、标题、摘要、元信息
- **UserCard**：用户卡片，包含头像、用户名、简介、关注按钮
- **CommentItem**：评论项，支持嵌套回复、点赞、时间戳
- **Tag**：标签组件，支持渐变边框和悬停效果
- **Statistic**：统计组件，用于展示数字数据

#### 业务组件层（Business Components）
**官网着陆页组件**（Landing Page Components）：
- **Navigation**：导航栏组件，包含logo、菜单、登录按钮、移动端菜单
- **HeroSection**：英雄区域，包含主标题、副标题、行动按钮、应用预览

- **PricingCard**：定价卡片，支持免费/付费计划对比

**文章相关组件**（Article Components）：
- **ArticleEditor**：富文本编辑器，支持Markdown、图片上传、代码块
- **ArticleList**：文章列表，支持分页、筛选、排序

**用户相关组件**（User Components）：

- **UserBadge**：用户徽章，标识认证用户、专家等身份

**互动组件**（Interaction Components）：
- **LikeButton**：点赞按钮，支持数字显示、已点赞状态
- **CommentSection**：评论区，支持发表评论、回复、点赞
- **ShareModal**：分享模态框，支持多平台分享、复制链接
- **FollowModal**：关注弹窗，推荐相关用户
- **NotificationItem**：通知项，支持已读/未读状态

**了解页面组件**（About Page Components）：
- **VisionMissionSection**：愿景使命区域，包含双卡片布局展示愿景和使命
- **TimelineSection**：发展历程时间线，支持垂直时间轴展示
- **DeveloperProfile**：开发者档案卡片，包含头像、简介、技能标签
- **AboutCard**：关于卡片，支持图标、标题、内容的标准化布局
- **TimelineItem**：时间线项目，包含日期、标题、描述、状态标识

#### 功能组件层（Feature Components）
**搜索组件**（Search Components）：
- **SearchBar**：搜索栏，支持实时搜索、搜索历史、热门搜索
- **FilterPanel**：筛选面板，支持多维度筛选条件
- **SearchResults**：搜索结果展示，支持分类展示
- **SearchSuggestion**：搜索建议，实时显示搜索建议词

**导航组件**（Navigation Components）：
- **Breadcrumb**：面包屑导航，显示当前页面位置
- **Pagination**：分页组件，支持页码跳转、每页数量选择
- **Tabs**：标签页组件，支持横向/纵向布局
- **Dropdown**：下拉菜单，支持单选/多选、搜索
- **Sidebar**：侧边栏，支持折叠、图标菜单

**反馈组件**（Feedback Components）：
- **LoadingSpinner**：加载动画，支持不同尺寸和颜色
- **ErrorBoundary**：错误边界，捕获并显示组件错误
- **EmptyState**：空状态展示，提供友好的无数据提示
- **Skeleton**：骨架屏，提供加载时的占位效果
- **ProgressBar**：进度条，支持 determinate/indeterminate 状态

**三栏式主应用布局**（Three-Column Main Application Layout）：

#### 布局复用模式

**1. 共享布局模式**（Shared Layout Pattern）：
- **MainLayout**：作为所有主应用页面的基础布局容器
- **LeftSidebar**：在首页、探索、发布页面中完全复用
- **RightSidebar**：在首页和探索页面中复用（发布页面隐藏）
- **布局一致性**：确保所有页面具有相同的视觉结构和交互体验

**2. 条件渲染模式**（Conditional Rendering Pattern）：
```typescript
// 布局组件中的条件渲染逻辑
interface LayoutProps {
  showRightSidebar?: boolean;      // 是否显示右侧边栏
  pageTitle?: string;              // 页面标题
  pageActions?: ReactNode;         // 页面操作按钮
  children: ReactNode;             // 主内容
}

const MainLayout: React.FC<LayoutProps> = ({
  showRightSidebar = true,
  pageTitle,
  pageActions,
  children
}) => {
  return (
    <div className="main-layout">
      <LeftSidebar />
      <div className="content-area">
        <TopNavigation title={pageTitle} actions={pageActions} />
        <main className="main-content">
          {children}
        </main>
      </div>
      {showRightSidebar && <RightSidebar />}
    </div>
  );
};
```

**3. 响应式布局模式**（Responsive Layout Pattern）：
```typescript
// 响应式布局断点配置
const LAYOUT_BREAKPOINTS = {
  mobile: '(max-width: 640px)',      // 移动端：隐藏侧边栏，显示底部导航
  tablet: '(max-width: 1024px)',    // 平板端：显示图标模式侧边栏
  desktop: '(min-width: 1025px)',   // 桌面端：显示完整三栏布局
};

// 布局状态管理
interface LayoutState {
  sidebarCollapsed: boolean;          // 侧边栏折叠状态
  rightSidebarVisible: boolean;       // 右侧边栏显示状态
  isMobile: boolean;                  // 是否为移动端
  isTablet: boolean;                  // 是否为平板端
}
```
